### Phase 2: ガチャシステム（優先度: 高）

#### ステップアップコスト
| 回数 | コスト | SR率 |
|---|---|---|
| 1回目 | 30P | 3% |
| 2回目 | 60P | 3% |
| 3回目 | 100P | 5% |
| 4回目 | 150P | 5% |
| 5回目 | 300P | 8% |
| 6回目 | 500P | 天井（SR確定） |
| 7回目〜 | 500P | 10%、6回ごと天井 |

#### 排出
- **N（70%）**: 通常ランダム住民（AddNPC相当）
- **R（25%）**: パラメータが極端に尖った住民
- **SR**: 専用ビジュアル＋特殊職業の組み合わせ

#### SR定義
- SR = 職業 × 服装 × 髪型のマスター定義
- 所持済みSRは排出テーブルから除外
- 全SR所持時は天井がR確定（UI明記）
- SR職業例: 神託の巫女、預言者、錬金術師、風来の剣士

#### ガチャの種類
- **キャラガチャ**: 物理P専用（200P/回 → ステップアップに変更済み）
- **天変地異ガチャ**: 概念P専用（チケット制、任意タイミング使用）
- **信仰P**: どちらにも使える万能通貨（300P/回、割高）

#### 天変地異ガチャ（イベント召喚）
- チケットとして保持、任意タイミングで使用
- イベント例: 火災、飢饉、隕石、謎の旅人来訪、大洪水
- 排出率は高め（ストレスにならない設計）
- 「いつ使うか」の判断がゲーム性

## スプライトシステム仕様

### フォルダ構成
```
public/sprites/
  f/          ← 女性パーツ
  m/          ← 男性パーツ
  job/        ← 職業装飾（男女共通）
```

### 命名規則
```
{パーツ}_{性別}_{番号}.png        通常版
{パーツ}_{性別}_{番号}_p.png      プロフィール版（太アウトライン）
{パーツ}_{性別}_{番号}_high.png   ハイライト
job_{職業ID}_{パーツ名}.png       職業装飾
job_{職業ID}_{パーツ名}_p.png     職業装飾プロフィール版
```

### レイヤー順（下→上）
**女性:** hair_back → body → cloth → [job_robe] → eye → mouth → hair_front → eye_high → hair_front_high → [job_hat] → [job_ribbon/necklace]
**男性:** body → cloth → [job_robe] → eye → mouth → [beard] → hair_front → eye_high → hair_front_high → [job_hat] → [job_ribbon/necklace]

### 色管理
- 髪色: npc.color（テーマカラー）、CSS filter darken=0.7
- 服色: IDハッシュ生成、CSS filter darken=0.65
- 肌色: 6色パレット、専用skinToFilter（黄色み補正あり）
- 目色: 6色パレット
- 口色: 赤〜ピンク系ハッシュ生成
- ひげ色: 髪色同系、darken=0.6
- 職業装飾: color指定ありはCSS filter、なし(null)はそのまま

### 職業装飾マッピング（CharacterSprite.tsx JOB_LAYERS）
```typescript
'農民': [hat(top,null), ribbon(top,hair)]
'聖職者': [robe(over_cloth,cloth), necklace(top,null)]
'僧侶': [robe(over_cloth,cloth), necklace(top,null)]
```
新しい職業装飾を追加する時:
1. public/sprites/job/ にPNG追加（+_p版）
2. CharacterSprite.tsx の JOB_LAYERS に定義追加

### 初期キャラオーバーライド（NPCSprite.tsx INITIAL_OVERRIDES）
```
takeshi: hairFront='03'(スキンヘッド), hasBeard=true
genji: hairFront='02', hasBeard=true
sora: hairFront='01', hasBeard=false
```

### カラーパレット（AddNPCModal.tsx COLORS）
100色。赤/オレンジ/茶/黄緑/緑/青/紺/紫/ピンク/グレー/ゴールド各系統に複数バリエーション。

## ささやきシステム仕様

### 表現ランダム化（promptBuilder.ts WHISPER_EXPRESSIONS）
「不思議な声」「夢のお告げ」「胸騒ぎ」「天候の兆し」「偶然の印」「風のささやき」
→ UIログは「神の声」のまま

### 職業×パラメータ別解釈（getWhisperInterpretation）
- 聖職者/僧侶(道徳+共感≥40): 神の啓示として
- 職人/鍛冶師(論理≥20): 技術課題として
- 商人(野心≥20): 商機として
- 探検家(創造≥20): 発見の手がかりとして
- 農民: 土地・水・収穫の教えとして
- 漁師/猟師: 自然の知らせとして
- 薬師: 癒しの知恵として
- 楽師/語り部: 物語の着想として
- **聖職者以外は「宗教的解釈は最後の手段」が入る**

### 介入副作用（whisperIntensity）
- 全NPCの長期記憶中の介入関連記録の割合で計算
- 0.3以上: 「何かに導かれるような出来事が続いている」
- 0.5以上: 「聖職者が利用、商人が不安を商機に、懐疑的な者も」
- 0.8以上: 「依存・疑念・反発。従う者と疑う者の対立」

## 祈りシステム仕様

### 生成
- 3日ごとにAI(Llama-8B)で1〜2件生成
- 未成就の祈りが3件以上あれば生成しない
- NPCの気分・信念に基づいた素朴な願い
- 各祈りにkeywords（場所名・物の名前）付き

### 成就判定
- civilization_event発生時にkeywordsとeventのname+descriptionをマッチ
- マッチしたらAI(Llama-8B)に「叶ったか？」を1回だけ聞く
- 成就 → 信仰P +10、祈り一覧で「成就」表示

### 偶然の成就
- 神が何もしなくても、自然発生のイベントが祈りと一致すれば成就
- 住民は「神が叶えてくれた」と解釈する可能性（AI任せ）
- 宗教の発生メカニズムのシミュレーション
